# Project Context

## Role
Senior-level backend Firebase developer prototyping a secure API for a prospective client.

## Client
**SageReport** - https://sagereport.com/

## Situation
- Client has already hired a UI/UX developer
- Client is evaluating whether to add a backend developer
- This project serves as a demo for the CTO to demonstrate backend value

## Objective
Create a sample project with real endpoints and authentication that showcases:

1. **Security** - Authentication, authorization, input validation, secure data handling
2. **Logging** - Comprehensive logging for debugging and audit trails
3. **Webjobs** - Scheduled tasks, background processing, automation
4. **Monitoring** - Health checks, performance metrics, alerting
5. **Scalability** - Architecture that grows with the business
6. **Profitability** - Cost-effective solutions, efficiency gains, ROI justification

## Tech Stack
- Firebase (Authentication, Firestore, Cloud Functions, Hosting)
- Node.js backend

## Goal
Impress the CTO and secure the backend developer role by demonstrating clear value-add over a frontend-only approach.
